## 🎓 DEVELOPER GUIDE

### Understanding the Flow

1. **Client** navigates to photographer profile
2. **Client** clicks "Request Booking"
3. **System** displays booking form (Create.vue)
4. **Client** fills details and submits
5. **System** creates BookingRequest record
6. **System** logs initial status
7. **System** sends notification to photographer
8. **Photographer** receives email notification
9. **Photographer** views booking in inbox
10. **Photographer** accepts/declines via Show.vue
11. **System** updates status and logs change
12. **System** notifies client
13. **Both parties** can exchange messages
14. **Photographer** marks booking complete
15. **Client** receives completion notification
16. **Admin** can monitor all bookings

### Key Files to Know

**For Adding Features:**
- `app/Models/BookingRequest.php` - Add new relationships/scopes here
- `app/Http/Controllers/BookingRequestController.php` - Add new actions here
- `app/Policies/BookingRequestPolicy.php` - Add new authorization gates here

**For UI Changes:**
- `resources/js/Pages/Bookings/Show.vue` - Booking detail page
- `resources/js/Pages/Bookings/Create.vue` - Booking form
- `resources/js/Pages/Admin/Bookings/Show.vue` - Admin dashboard

**For Database Changes:**
- Create new migration: `php artisan make:migration`
- Add relationship to model
- Update policy if needed
